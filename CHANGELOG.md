### Changed

- Lighthouse CI's accessibility gate (`lighthouserc.json`) moved from a flat sitewide `warn` to a per-page `assertMatrix`: `error` at `minScore: 0.95` on the 7 tool pages, `warn` at the same threshold on `404.html`, which carries one confirmed `color-contrast` false positive (its `aria-hidden` decorative watermark, redundant to real error text) documented as an accepted exception in ADR-0005 (ticket T-20).
- Self-hosted the Phosphor icon webfont (regular + duotone weights) under `assets/phosphor/` on all 8 pages, replacing the synchronous `<script src="https://unpkg.com/@phosphor-icons/web@2.1.1/src/index.js">` with two local `<link rel="stylesheet">` tags. Removes a render-blocking, unminified third-party script and its unpkg SPOF risk (audit finding F-02 / ticket T-02).
- `lookup.html`'s `<title>` and `og:title` changed from "Stroke & ED Throughput · Q&A Bank" to "Q&A Lookup — AbstractionDeskQA" — the old title had no brand name and was stale relative to the page's actual scope, which now includes SEP-1/HBIPS content too (audit finding F-08).
- `Sawyer-1.jpeg` (487 KB, no dimensions) on `404.html` replaced with `Sawyer-1.webp` (600×450, ~40 KB) with explicit `width`/`height` attributes on the `<img>`; original JPEG removed from the repo (audit finding F-10).
- Darkened the shared `--ink-soft` token (`#6b7f94` → `#5a6d81`, on `index.html`, `sep1-tool.html`, `hbips-tool.html`, `abstractly.html`, `404.html`, plus the same hardcoded value in `lkw-tool.html`/`cmo-tool.html`) and lookup.html's `--amber` token (`#c27a00` → `#a36600`) to clear WCAG AA 4.5:1 contrast — both previously failed (3.91:1 and 3.46:1 respectively) (audit finding F-13).
- `lookup.html`'s data read path (ticket T-07): cold load and background refresh now fetch the same-origin `data/qa.json` instead of calling Apps Script live; `fetchFromSheets()` renamed `loadQaData()` since it no longer talks to Sheets, and `CACHE_TTL` raised from 5 minutes to 12 hours since the backing file only changes nightly. Write/submission paths (`postJSON()` to `SCRIPT_URL`) are unchanged.
- CI baseline config: `.html-validate.json` now sets `void-style` to `error` (self-closing void elements, e.g. `<br />`, enforced site-wide) alongside the existing `extends` and `lookup.html` heading-level override, both left unchanged; `lighthouserc.json`'s `categories:performance` assertion downgraded from `error` to `warn` (`minScore: 0.9` unchanged), `categories:accessibility` untouched. No ticket currently owns re-tightening the performance gate back to error.
- `index.html`, `sep1-tool.html`, `hbips-tool.html`, `abstractly.html`, `lkw-tool.html`, `cmo-tool.html` now link the shared `tokens.css`/`site.css`/`site.js` instead of maintaining local copies of nav/footer/disclaimer/share-strip/skip-link/focus-visible CSS (ticket T-04). Two deliberate exceptions: `index.html`'s nav stays local (a different design serving in-page scroll anchors, not cross-tool navigation); `lookup.html`'s typography and navy/teal/red/amber/green palette stay local (only the canonical purple accent and generic structural tokens converge).
- Canonical purple accent `#7B5CE5` replaces the two divergent purple hexes previously in use (`cmo-tool.html`'s `#a78bfa`, `lookup.html`'s `#5430a8`/`#7552d8`) — teal remains the primary brand/CTA color, unchanged.

- `hbips-tool.html`'s monospace token converges from `Courier New` (system font) to `DM Mono` (the site-wide value), visible on the calculator's result numbers and duration inputs.

### Fixed

- Keyboard-operable case-cards and help buttons via a shared delegated handler (ticket T-18, closes TD-17): `hbips-tool.html`'s 6 `.help-btn` spans (`role="button" tabindex="0"` but no keydown handler at all) and the `.case-card` activation elements on `sep1-tool.html`/`hbips-tool.html` (previously Enter-only) and `lkw-tool.html`/`cmo-tool.html` (previously not even focusable) now all respond to both Enter and Space, via one new delegated `keydown` listener in `assets/site.js` rather than per-element inline handlers. Deliberately excludes `cmo-tool.html`'s doc-type toggles and `sep1-tool.html`'s type-picker widget — those need checkbox/popup ARIA semantics, not plain button activation, and are split to a new ticket, T-19.
- `lkw-tool.html` and `cmo-tool.html` tab navs now implement the full ARIA tabs pattern instead of a bare `role="tablist"` with no follow-through: each tab button gets `role="tab"`, `aria-selected`, `aria-controls`, and roving `tabindex`, each panel gets `role="tabpanel"`/`aria-labelledby`, and `ArrowLeft`/`ArrowRight`/`Home`/`End` now move focus between tabs (audit finding F-11 / ticket T-10).
- `sep1-tool.html`, `lkw-tool.html`, and `cmo-tool.html` no longer skip `<h1>` straight to `<h3>`: each page's per-tab section label (previously a plain `<div>`) is now an `<h2>`, giving a proper `h1 → h2 → h3` structure with no visual change (ticket T-12). `.html-validate.json`'s `heading-level` rule now runs at `error` by default site-wide; `lookup.html` is carved out to `warn` via a scoped override since its own instance of this skip is unchanged here and remains tracked under T-05.
- Escaped raw `&` to `&amp;` in visible text content at 8 locations: `hbips-tool.html:1059`, `index.html:828`/`868`, `lkw-tool.html:848`, `cmo-tool.html:275`/`758`, and `lookup.html:7`/`1051` (the `Q&A`/`H&P`/`Q&As` copy across nav taglines, the newsletter blurb, the CMO decision-tree tile, and the `<title>` element). Found via a CI-baseline `&` audit; HTML comments, CSS comments, and attribute-value occurrences (`aria-label`, `meta[content]`) were left un-encoded on purpose — not html-validate-flagged and out of scope for this pass.
- Renamed the html-validate config `.html-validate.json` → `.htmlvalidate.json` (audit finding F-15 / ticket T-13): the hyphenated filename is not one of html-validate's auto-discovered config names, and `ci.yml`'s `npx html-validate` step passes no `--config` flag, so the entire config — `extends: html-validate:recommended`, the `lookup.html` heading-level override, and the `void-style` rule added in the CI-baseline pass above — was silently never loaded by CI. `void-style` enforcement was a no-op until this rename.
- T-13 follow-up, now tracked as **F-21** (F-15 was already assigned to the Sheets backend-fragility finding resolved by T-07 — corrected here, not rewritten above): the rename alone wasn't enough. `.htmlvalidate.json` also had a top-level `overrides` key, which isn't part of html-validate's config schema (only `extends`/`rules`/`elements`/`plugins`/`transform`/`root` are valid) — once the config actually loaded, that unrecognized key would have failed schema validation and made html-validate exit 1 before checking any file. Removed `overrides` entirely, and also removed the explicit `void-style` override, reverting to `recommended`'s own `error`/`"omit"` default (the codebase is already ~88% compliant with `"omit"`; the `"selfclosing"` override from the CI-baseline pass would have failed hard site-wide instead). The `lookup.html` heading-level carve-out `overrides` used to provide is now a single inline `<!-- html-validate-disable-next heading-level -->` directive immediately before the one specific offending `<h3>` (`lookup.html:301` — the document's first heading, with no `h1`/`h2` before it) rather than a file-wide config-level warn. `no-unused-disable` (enabled in `recommended`) will flag that directive as stale once T-05 gives the page a real heading structure — a self-cleaning reminder in place of a silent `warn`.
- Merged `lookup.html`'s `esc()` and `sep1-tool.html`'s `escH()` into one shared, quote-safe `esc()` in `assets/site.js` (audit finding F-18) — `escH()` previously did not escape single quotes.
- `abstractly.html`: cleared all 5 `no-inline-style` and 2 of its 4 `no-redundant-role` html-validate findings; the remaining 2 `no-redundant-role` hits live on `404.html`, out of this ticket's scope.
- `lookup.html` no longer skips straight from a missing `<h1>` to its first `<h3>` — added a visually-hidden `<h1>`/`<h2>` and removed the scoped `html-validate-disable-next heading-level` directive T-13 had left in place; `heading-level` now passes at `error` severity site-wide with no remaining per-page carve-out (ticket T-05).
- `cmo-tool.html`: 2 self-closing `<meta/>` tags (`charset`, `viewport`, lines 4-5) now match the site's `omit`-style convention (`<meta charset="UTF-8">`). T-13 believed `void-style` was already zero sitewide; T-04's page migration reintroduced this instance, hidden until now by a GitHub Actions annotation-cap issue (see the T-14 Phase A Why-Note in ARCHITECTURE.md) that was truncating every CI baseline check to 10 errors site-wide. Fixed as a tag-along in the T-14 Phase A commit.
- `lighthouserc.json`'s `assertMatrix` carve-out for `404.html` (added in T-20/ADR-0005) didn't actually exempt the page from the accessibility `error` gate: `assertMatrix` entries are additive, not override, so 404.html matched both the sitewide `error`/0.95 entry and its own `warn`/0.95 entry, and the `error` assertion still hard-failed CI. Restructured to three entries — the sitewide entry now carries only the performance assertion, a new entry with a negative-lookahead pattern (`^(?!.*404\.html).*`) carries the accessibility `error` assertion and structurally excludes 404.html, and the existing 404.html-scoped `warn` entry is unchanged (ticket T-21, closes TD-19, see ADR-0006).
- `no-implicit-input-type`: added explicit `type=` to 24 inputs missing one — 11 in `lookup.html` (all free-text filter/search/tag fields, `type="text"`), 13 in `lkw-tool.html` (10 date/time-grid inputs plus the military-time-converter and hours-ago-subtractor calculator inputs, `type="text"` since each is read as an opaque string or matched against a custom regex rather than a native date/time value; `calc-hrs` is `type="number" step="any"`, the one genuinely numeric field, read via `parseFloat()`) (ticket T-22).
- `no-trailing-whitespace`: removed a trailing-whitespace-only blank line in `lkw-tool.html:424` (ticket T-22).
- `prefer-native-element`: converted 3 unambiguous semantic wrapper elements to their native equivalents — `hbips-tool.html`'s and `sep1-tool.html`'s `<div class="page-hero" role="banner">` → `<header class="page-hero">`, and `index.html`'s `<div class="stats" role="complementary">` → `<aside class="stats">` — dropping the now-redundant explicit `role` attribute on each (ticket T-22).
- Native checkbox semantics for `cmo-tool.html`'s 8 doc-type toggles and a full ARIA menu pattern for `sep1-tool.html`'s type-picker (ticket T-19, closes the remainder of TD-17): both were previously mouse-only clickable `<div>`s with no `role`, `tabindex`, or keyboard handling at all, deliberately left out of T-18's plain-button fix because neither is a plain button — the cmo toggles are a multi-select checkbox group and the sep1 picker is a popup menu. cmo's toggles are now `<label>`-wrapped native `<input type="checkbox">` elements (visually hidden via `.sr-only`, reusing the same pattern as the page's own Timing Flags checkboxes); sep1's picker trigger is now a native `<button>` with `aria-haspopup`/`aria-expanded`/`aria-controls`, and its 5 options carry `role="menu"`/`role="menuitem"` with roving-tabindex Arrow/Home/End navigation plus Escape-to-close, mirroring the site's existing tablist keyboard pattern.

### Security

- Added the Cloudflare Turnstile widget to all 3 submission paths in `lookup.html` (add-entry panel, tag-manager add-tag, paste modal); each submission now requires a completed challenge token attached to the payload (audit finding F-03 / ticket T-03a — client-side half). **Not yet a real mitigation**: the Apps Script backend does not verify the token server-side, so this does not close F-03 on its own. Server-side verification is tracked as ticket T-11, blocked on Apps Script source living outside this repo.
- Fixed `esc()` in `lookup.html` to also escape `"` and `'` (previously escaped only `& < >`), closing the attribute-boundary XSS gap identified in audit finding F-01 / ticket T-01. Note: this closes the risk for all plain-text and non-JS-attribute uses of `esc()`; the inline `onclick="..."` sites still carry residual risk because browsers HTML-decode attribute values before compiling inline event-handler script, so an escaped `'` decodes back to a literal quote before the nested JS string is parsed. Full closure of that vector requires replacing inline `onclick` handlers with delegated `addEventListener` + `data-*` attributes (tracked separately, not part of T-01's scope).
- Added a heuristic PHI scrubber to the new nightly `data/qa.json` sync (ticket T-07): before writing the file, `scripts/sync-qa-data.mjs` scans every fetched entry for labeled identifiers (MRN/SSN/DOB/etc.), SSN-shaped strings, long digit runs, and birth/admission/discharge dates that sit within 30 characters of a PHI label — any match refuses the write/commit rather than auto-redacting, per ADR-0003. The nightly workflow may commit directly to `main`, but only `data/qa.json`; an automated scope guard fails the job if the diff ever touches any other file — see ADR-0004 for the full reasoning behind this scoped exception to the normal PR-only rule.
