## 1. System Overview

Static multi-page toolkit for hospital core measure abstractors. GitHub Pages hosting, custom domain (CNAME), no build step, no server-side code owned by this repo. One page (lookup.html) consumes an external Google Apps Script API backed by Google Sheets.

```
┌────────────────────────────── abstractiondeskqa.com (GitHub Pages) ─┐
│ index  lookup  sep1  lkw  cmo  hbips  abstractly  404               │
│   │       │                                                        │
│   │       ├─ GET data/qa.json (same-origin static) ──► SEED fallback│
│   │       ├─ POST submissions ──► Google Apps Script ──► Sheets    │
│   │       └─ cache: localStorage (12h TTL)                         │
│   └─ feedback POST ──► formly.email (+ Cloudflare Turnstile)       │
│ all pages ──► GA4 · Google Fonts · phosphor icons (self-hosted) · BMC│
│                                                                      │
│ nightly (CI): scripts/sync-qa-data.mjs ─► Apps Script ─► Sheets     │
│   → PHI scrubber → data/qa.json (bot commit, scoped, main)         │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. Data Flow — lookup.html

1. **Read (T-07):** Load → check `qc-sheets-cache-v1` (TTL 12h). Hit → render, refresh in background against `data/qa.json`. Miss → GET same-origin `data/qa.json` (static, regenerated nightly — see step 5).
2. Fetch failure or empty `entries` → `entries = [...SEED]` (embedded fallback), tag defaults merged, visible warning badge shown. There is intentionally no live-Sheets fallback tier between the static file and SEED — the whole point of T-07/TD-4 was to remove the read path's live-Sheets dependency, not add a rare-case one back in. **Known gap:** the background refresh's own fetch failures are still silent (no badge, no telemetry) — only the cold-load failure path is user-visible.
3. User submissions (add panel, paste modal) → client-side auto-tag detection → review chips → POST JSON to `SCRIPT_URL` → sheet row awaiting curator approval. **Known gap:** no bot protection or rate limit on this path (index form has Turnstile; this doesn't).
4. Render: full innerHTML template re-render; interactivity via inline onclick handlers calling globals.
5. **Nightly sync (T-07):** `.github/workflows/sync-qa-data.yml` (schedule + `workflow_dispatch`) runs `scripts/sync-qa-data.mjs`, which GETs the same two Apps Script endpoints (`SCRIPT_URL`, `SCRIPT_URL?type=tags`), runs a heuristic PHI scrubber over every entry, and writes `data/qa.json` (`{entries, tags, generatedAt}`). The workflow verifies the diff touches only `data/qa.json` before committing as `github-actions[bot]` directly to `main` — see ADR-0004 for the scoped exception to the normal PR-only rule.

## 5. Cross-Cutting Conventions (target state)

- **Output encoding:** single shared `esc()` escaping `& < > " '`, now living in `assets/site.js` (T-04 merged lookup.html's `esc()` and sep1-tool.html's `escH()` — the latter never escaped single quotes — closing F-18). Every page's dynamic-value escaping calls this one function.
- **Design tokens:** `assets/tokens.css` holds a light default `:root` (index/sep1-tool/hbips-tool/abstractly, plus lookup.html's structural/purple subset) and a `:root[data-theme="dark"]` override (lkw-tool/cmo-tool) (T-04). Two deliberate, documented carve-outs remain — not drift: **index.html's nav** stays page-local (a different design serving in-page scroll anchors, not cross-tool links — reskinning it would misrepresent its own navigation model); **lookup.html's typography and navy/teal/red/amber/green palette** stay page-local (only the canonical `--purple:#7B5CE5` accent and generic structural tokens converge — see the 2026-07-07 Why-This-Change note for the full reasoning).
- **Breakpoints:** 640px (mobile), 900px (tablet). (Baseline: 6 ad-hoc values.)
- **Events:** delegated `addEventListener`, no new inline `onclick`. T-04 advanced this in a bounded way: the tab-widget (`switchTab`/`showTab`) and share-strip copy-link are now delegated via `site.js`. The bulk of onclick handlers — lookup.html's ~55 Q&A/tag/card handlers, and each tool page's spec-logic UI wiring (`gAns`, `toggleHelp`, quiz/case navigation, etc.) — are unchanged, out of this ticket's scope by design, not an oversight.
- **Script load order:** `assets/site.js` is loaded synchronously (`<script src="assets/site.js"></script>`, no `defer`), placed immediately before each page's own inline `<script>` block — never a deferred `<head>` reference. Reason: some pages (e.g. sep1-tool.html's seed-data loop) call `esc()` synchronously at parse time, before `DOMContentLoaded`; a deferred site.js would not yet be loaded when that code runs, causing a `ReferenceError`. One uniform placement rule avoids a per-page judgment call each time a new page is added.
- **Spec logic:** pure functions in `assets/spec-logic/`, unit-tested against manual-cited cases; page files only wire UI.

## 7. ADR Index

- **[ADR-0001 — Single-file-per-tool, no build step](docs/adr/0001-single-file-per-tool-no-build-step.md).** *Accepted (retroactive).* Context: solo builder, GitHub Pages, tools must be trivially copyable/shareable. Consequence: duplication managed via shared `<link>`/`<script>` assets rather than a bundler; revisit if pages exceed ~10 or a framework need emerges.
- **[ADR-0002 — Google Sheets as content CMS](docs/adr/0002-google-sheets-as-content-cms.md).** *Accepted, amendment adopted (T-07).* Context: curator (non-dev) edits content; zero backend cost. Amendment: nightly export to versioned static JSON, Sheets remains authoring surface (fixes latency/quota, adds provenance/diffability) — implemented per ADR-0004.
- **[ADR-0003 — PHI-free by design](docs/adr/0003-phi-free-by-design.md).** *Accepted.* No user chart data is transmitted or stored; tools operate on user-entered abstractions client-side only. Any future feature that would transmit clinical text requires a new ADR.
- **[ADR-0004 — Static JSON pipeline with scoped bot-commit exception](docs/adr/0004-static-json-pipeline-and-scoped-bot-commit.md).** *Accepted.* lookup.html reads a nightly-regenerated `data/qa.json` instead of calling Apps Script live; the nightly workflow may commit directly to `main`, but only that one file, enforced by an automated scope guard, plus a PHI-pattern scrubber before every write.
- ADR template: `docs/adr/NNNN-title.md` → Context / Decision / Status / Consequences / Alternatives considered.
